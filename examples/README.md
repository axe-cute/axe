# Examples

This directory contains standalone examples demonstrating axe's capabilities.

## 🎯 Standalone Package Examples

These show individual `pkg/` packages used in isolation (no framework needed):

| # | Name | Package | Description |
|---|---|---|---|
| 01 | [Error Taxonomy](01-apperror/) | `pkg/apperror` | Typed HTTP errors with consistent JSON responses |
| 02 | [Transactions](02-txmanager/) | `pkg/txmanager` | Unit of Work pattern — no more manual `*sql.Tx` passing |
| 03 | [Plugin System](03-plugin/) | `pkg/plugin` | Build and register a custom plugin with dependency injection |

```bash
go run ./examples/01-apperror
```

## 🏗️ Full Production Projects

These are **real, runnable API projects** generated by `axe new` + `axe generate resource`:

| Project | Domain | Resources | Key Features |
|---|---|---|---|
| [E-Commerce](ecommerce/) | 🛒 Online store | Product, Order, Review | JWT auth, parent-child (Review→Product), multi-resource auto-wiring |
| [Webtoon](webtoon/) | 📖 Manhwa platform | Series, Episode, Bookmark | UUID keys, parent-child (Episode→Series), user bookmarks |

```bash
cd examples/ecommerce
docker-compose up -d
cp .env.example .env
make migrate-up
make run
```

### What these prove

1. **`axe new` → working project in < 2 minutes**
2. **`axe generate resource` × 3 → 27 files generated, auto-wired, compiled**
3. **Real domain relationships** (`--belongs-to`, `--with-auth`)
4. **Production infrastructure** (Docker, migrations, Redis, Prometheus metrics)
